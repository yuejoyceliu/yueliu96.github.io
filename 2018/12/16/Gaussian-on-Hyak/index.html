Gaussian on Hyak

Posted on 2018-12-16 | In Turecek Lab Tutorial

Here introduces how to setup Gaussian input file, submit it to Hyak (super computer server) and analyze the output file by my python scripts.

Gaussian 16 Input

Gaussian 16 input consists of a series of lines in an ASCII text file. In general, the input is free-format and case-insensitive; comments start with an exclamation point(!), could be anywhere on a line; spaces, commas, tabs and forward slashes are all considered as valid delimiters (multi spaces are treated as a single demiliter); keyword=option,kw(op),kw=(op1,op2,…) and kw(op1,op2) are all correct. The common suffix of the input file is gjf or com, but com file cannot be sent by gmail. The basic structure of a Gaussian input file includes several different sections:

  • Link 0 Commands

    • locate and name scratch files
    • not blank line terminated
    • examples
      • %LindaWorker
        • add it when using 2 or more nodes on Hyak
      • %NProcShared=N
        • use up to N processors/cores on shared memory on one node
        • use a smaller number when have memory issue
        • default: 28 on Hyak-mox, 16 on Hyak-ikt
      • %UseSSH
        • start linda workers by ssh instead of rsh(default)
        • rsh, remote shell, can execute command on another computer as another user
        • ssh: a more secure way to communicate between computers
        • some parallel jobs may mess up by using rsh
      • %Mem=N
        • set the dynamics of memory, can follow by MB,GB,etc.
        • 800mb for g16 by default if not specify
        • some gaussian jobs will select the appropriate algorithm automatically based on the setting of %Mem and MaxDisk.
        • faster than disk operations by many orders of magnitude.
        • specifing more mem than availble one will cause very poor performance, suggest use 90% of max mem
      • %RWF=file
        • add it if want to store read-write file, a super huge file
        • usually followed by %NoSave, everything before nosave will deleted if job completes normally
        • useful for restart a job; restart opt only needs chk file
        • a suggest location on Hyak to store it is /gscratch/scrubbed/
        • how to locate it if not include this line (Hyak version):
          • all scratch files are at /scr/ of the local node, will be deleted after walltime runs out (find node number run ssh nodenumber to enter that node)
          • Related key word: MaxDisk
      • %Chk=file
        • add it to locate and name the checkpoint file
        • useful, suggest to keep it
        • can restart a job and visualize molecular orbitals for tddft
      • %OldChk=file
        • use with %Chk=newfile: copy file to newfile and then write new chk info to newfile

  • Route section (# lines)

    • job type and method
    • blank line terminated
    • examples
      • print form
        • #N: normal print, by default if not specify
        • #P: print more info
        • #T: terse print
      • job type
        • SP : single point energy, by default if not specify
        • OPT : optimization
        • FREQ : vibrational frequency and thermal infomation
        • TD(NStates=n) : calculate n excited states
      • other keywords
        • POP
          • charge and spin distribution
          • default except guess=only: pop=min
        • SCF
          • controls the functioning of the SCF procedure
          • scf=tight by default
          • scf=xqc is helpful for difficult conversion case
          • NoSymm or Symmetry=None: release orbital symmetry constraints
        • SCRF
          • in the presence of solvent, place solute in a cavity within the solvent reaction field.
          • scrf=pcm by default
          • ex: scrf=(pcm,solvent=water)
        • CacheSize
          • the amount of cache per processor to use with various cache-blocking algorithms (in 8-byte words)
          • from /proc/cpuinfo find the cache available(35840kb)
          • a good value is its 50%: $35840\times1024\div8\times50\%=2293760$: cachesize=2293760

        • MaxDisk
          • the amount of disk storage available for scratch data
          • disk space of one node on Hyak is ~100GB. lsblk -d or df -h can show physical disk info of linux system
          • the more disk space available, the faster the evaluation, especially for MP2. Some jobs have fixed or minimum disk requirments, click link for more info.

  • Title section

    • any descriptions
    • cannot exceed 5 lines
    • avoid special character: @ # ! – _ \ control characters (especially Ctrl-G)
    • blank line terminated

  • Molecule specification

    • charge, multiplicity, atoms and coordinates
      • multiplicity
        • number of possible spin orentations of the total spin
        • mp=2S+1, S($\alpha$ electron)=0.5, S($\beta$ electron)=-0.5
    • blank line terminated

  • Optinal additional sections

    • additional input
    • blank line terminated
